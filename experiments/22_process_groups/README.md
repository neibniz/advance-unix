# 实验 22：进程组、会话与子进程回收

## 目标

让 `fork` 出的子进程通过 `setsid` 创建新会话，验证它同时成为会话首进程和进程组首进程，并由父进程可靠接收结果和回收子进程。

## 关键 API

- `fork(2)`：创建子进程。
- `setsid(2)`：创建新会话和新进程组。
- `getpgid(2)`、`getsid(2)`：查询进程组 ID 与会话 ID。
- `pipe(2)`：把子进程观察到的 PID、PGID、SID 报告给父进程。
- `waitpid(2)`：等待并回收指定子进程。

## 原理

子进程继承父进程所在的进程组，但自身不是该组首进程，因此可调用 `setsid`。成功后应满足 `PID == PGID == SID`。子进程把三个值和错误状态通过管道完整写给父进程；父进程处理短读和 `EINTR`，独立校验报告及退出状态，最后用 `waitpid` 确认子进程已被回收。

## 构建与运行

```sh
cmake --preset dev
cmake --build --preset dev --target lab_process_groups
./build/dev/bin/lab_process_groups
```

## 预期输出

```text
child pid=1234 pgid=1234 sid=1234; waitpid reaped it
```

PID 每次运行都可能不同。

## 注意事项

- 已经是进程组首进程的调用者不能成功执行 `setsid`；常见守护进程流程先 `fork`，再由子进程调用它。
- 新会话起初没有控制终端；如何取得控制终端是独立的终端管理主题。
- 进程组用于信号分发和作业控制，会话则把一个或多个进程组组织在一起。
- 管道两端在父子进程中都应尽早关闭不用的副本，否则读端可能无法观察到 EOF。
