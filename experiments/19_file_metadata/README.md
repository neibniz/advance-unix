# 实验 19：文件元数据、权限与时间戳

## 目标

在私有临时目录中安全创建文件，观察 `umask` 对初始权限的影响，再通过文件描述符修改权限和时间戳，并用 `fstat` 自验证所有结果。

## 关键 API

- `mkdtemp(3)`：创建权限为 `0700` 的唯一临时目录。
- `openat(2)`：相对于已打开目录原子创建文件，避免依赖当前工作目录。
- `umask(2)`、`fchmod(2)`：设置权限屏蔽字和修改已打开文件的权限。
- `futimens(3)`：以纳秒形式设置访问时间和修改时间。
- `fstat(2)`：通过文件描述符读取类型、权限和时间戳。

## 原理

程序先用 `mkdtemp` 建立私有目录，再把进程的 `umask` 临时设为 `0027`。`openat` 请求模式 `0666`；没有默认 ACL 时，屏蔽组写和全部其他用户权限后得到 `0640`。程序记录实际初始模式，确认它没有超出请求权限，随后立即恢复并复查原 `umask`。接着用 `fchmod` 把权限确定为 `0600`，用 `futimens` 写入两个时间值，最后用 `fstat` 验证。无论在哪条退出路径，都会恢复进程状态、关闭描述符并删除文件和目录。

## 预期输出

```text
umask created mode=640; final mode=600 atime=1700000000.123456789 mtime=1700000123.987654321; umask restored
```

## 注意事项

- `umask` 是进程级状态，多线程程序中临时修改它会影响其他线程；真实服务应避免用这种方式做并发文件创建。
- 若父目录带有默认 POSIX ACL，创建模式由 ACL 继承规则共同决定，不一定严格等于 `0666 & ~0027`；输出会显示实际模式，固定 `/data` 环境的预期值为 `0640`。
- 私有目录限制了名称空间访问，`openat` 的 `O_CREAT | O_EXCL` 又保证已有同名文件不会被误用。
- 某些旧文件系统会降低时间戳精度；本实验要求底层文件系统能保存所设置的纳秒值。
- 权限比较只取 `st_mode` 的低权限位，文件类型则用 `S_ISREG` 判断。
